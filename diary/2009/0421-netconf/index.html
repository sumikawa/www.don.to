2009/04/21: netconfで遊んでみる その2

注: このperlスクリプトは個人的趣味で書いたものです。会社として動作を保証するものではありません。

その1の続き。 コンフィグの履歴管理をしたくなったので、config2git.plを作ってみました。

事前準備としてのスイッチ側の設定はその1を参照のこと。

また、githubから持って来たconfig2git.plを適当なパスにおいてください。 同じディレクトリにあるnetconf.plが必要なので、これもライブラリパスが通ったところに置くこと。 以上で準備は完了。

次に、config2git.plの最初の設定は自分の環境に合わせて書き換えてください。 @targetsが管理したいスイッチのIPアドレス。

my @targets = ( "192.168.2.3", "192.168.1.222", "192.168.1.223", );
my $gitdir = "/Users/sumikawa/configs";

後は、$gitdir 上で、"git init"を実行してください。 (.git ディレクトリが存在しなければ自動実行にしようかと思ったけど、勝手にinitやられるのは嫌だと思うので、やめました。)

で、config2git.plを実行。

% config2git.pl
Conect to 192.168.2.3
Conect to 192.168.1.222
Conect to 192.168.1.223
Commting...
[master (root-commit)]: created 3d9ab85: "20090421_03109"
 3 files changed, 561 insertions(+), 0 deletions(-)
 create mode 100644 192.168.1.222.txt
 create mode 100644 192.168.1.223.txt
 create mode 100644 192.168.2.3.txt
Finish.

$gitdir に"IPアドレス.txt"というファイルが置かれてるかと思います。 cronでも手動でもいいので、config2git.plを実行すれば、 git commitが自動実行されていきます。

差分の確認は、"git log -p"がいいかと思います。
